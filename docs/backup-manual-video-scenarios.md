# 月次メンテナンス動画 シナリオ集 v01(3本立て・各2〜3分)

> サイトまるごとバックアップパック(16,500円税込)付属の動画マニュアル用台本。
> 紙のマニュアル(`backup-manual-print.html` / `backup-manual.pdf` v01)の作業1〜3と1対1で対応。
> 制作方法: NotebookLM等のAI生成 or 画面収録+このナレーション読み上げ。
> 最終更新: 2026-07-22

---

## 動画タイトル(3本共通の付け方)

クライアントが「今月はどれを見ればいいか」で迷わないよう、**番号+作業名+ひとこと**で統一します。

| # | 動画タイトル | 長さ目安 |
|---|---|---|
| 1 | 【月イチ15分メンテ①】バックアップの確認 〜ちゃんと動いてる?を1分で見る〜 | 2分〜2分半 |
| 2 | 【月イチ15分メンテ②】更新(アップデート) 〜壊さない順番で最新にする〜 | 2分半〜3分 |
| 3 | 【月イチ15分メンテ③】サイトの健康チェック 〜お客様の目線で最終確認〜 | 2分〜2分半 |

**サムネイル文字案**: ①「バックアップ確認」②「安全に更新」③「健康チェック」+ 大きく番号。3本で色をそろえる(ブルー#00A2DE基調・番号だけオレンジ#F6AC0E)。

## 全編共通のトーン・ルール

- 呼びかけは「〜してください」ではなく「〜しましょう」。専門用語は使わない(紙マニュアルと同じ言葉づかいに合わせる)
- **各動画の冒頭あいさつは1文だけ**(長い自己紹介は入れない)
- **締めの定型文**(3本共通):
  > 「少しでも迷ったら、無理に進めず作業を止めて大丈夫です。LINE・メール・ショートメールで、いつでもご相談ください。ご相談は無料です。」
- 危険を伝える場面(作業2の失敗・作業3の乗っ取りサイン)は、**怖がらせず「止めて連絡すればOK」で着地**させる
- 画面収録する場合は**デモ用サイトの管理画面**を使う(実在のお客様サイト・個人情報は映さない)

## 動画① バックアップの確認(2分〜2分半)

**この動画のゴール**: 「最新バックアップの日付を見る」だけでいいと分かること。

| シーン | 画面に映すもの | ナレーション |
|---|---|---|
| オープニング(15秒) | タイトルカード→チェックリストの「作業1」 | こんにちは、ティーチ&ファネルDesignです。月イチ15分メンテ、1本目は「バックアップの確認」。やることはとてもシンプル、「いちばん新しいバックアップの日付を見る」——これだけです。 |
| ステップ1(20秒) | WordPressログイン画面 | まず、WordPressの管理画面にログインしましょう。ログインする場所は、お渡しした紙のマニュアルの巻末、「あなたのサイト情報」に書いてあります。 |
| ステップ2(20秒) | 管理画面→バックアップ一覧を開く操作 | 次に、バックアップの一覧画面を開きます。場所は納品のときにご案内したメニューです。これも巻末のメモに書いてあるので、迷ったらそちらを見てください。 |
| ステップ3(30秒) | バックアップ一覧の日付部分をズーム | 開いたら、見るのは1ヶ所だけ。いちばん新しいバックアップの「日付」です。ここ1週間以内の日付になっていれば、自動バックアップは正常に動いています。今月の確認はこれで合格です。 |
| ステップ4(25秒) | ダウンロード操作→PCのフォルダに保存 | 余裕があれば、もうひと手間。最新のバックアップを、ご自分のパソコンにもダウンロードしておきましょう。サーバーの外にも控えがあると、もしものときにさらに安心です。 |
| 注意(20秒) | 「1ヶ月以上前」の日付の例(赤枠) | もし日付が1ヶ月以上前で止まっていたら、自動バックアップが止まっているサインです。そのままにせず、私たちにお知らせください。 |
| 締め(15秒) | 連絡先カード(LINE・メール・SMS) | (共通の締め定型文) |

## 動画② 更新(アップデート)(2分半〜3分)

**この動画のゴール**: 「バックアップ確認→1つずつ更新→都度表示チェック」の順番が体に入ること。

| シーン | 画面に映すもの | ナレーション |
|---|---|---|
| オープニング(15秒) | タイトルカード→「作業1→作業2」の順番図 | 2本目は「更新」です。始める前にひとつだけ約束があります。必ず、1本目の「バックアップの確認」を先に済ませてください。バックアップさえあれば、更新で何が起きても元に戻せます。 |
| ステップ1(20秒) | ダッシュボード→「更新」画面 | 管理画面の左メニューから、「ダッシュボード」の中の「更新」を開きます。更新できるものがあると、数字のついたマークが出ています。 |
| ステップ2(40秒) | プラグインを1つだけ選んで更新する操作 | ここがいちばんのポイント。プラグインは、まとめて全部ではなく「1つずつ」更新しましょう。1つ更新したら、別のタブでサイトのトップページを開いて、見た目が崩れていないかをサッと確認。これを繰り返します。ひと手間に見えますが、もし不具合が出たとき「どの更新が原因か」がすぐ分かる、いちばん安全なやり方です。 |
| ステップ3(25秒) | テーマ更新→WordPress本体更新の操作 | プラグインが終わったら、テーマ、最後にWordPress本体の順で更新します。ここでも、更新のたびにトップページの確認を忘れずに。 |
| 注意(30秒) | 「大型アップデート」通知の例/真っ白画面のイメージ | 「メジャーアップデート」のような大きな更新の案内が出て不安なとき、または更新後に表示が崩れたり真っ白になったときは、慌てなくて大丈夫。バックアップから元に戻せます。操作をそこで止めて、そのままご連絡ください。 |
| 締め(15秒) | 連絡先カード | (共通の締め定型文) |

## 動画③ サイトの健康チェック(2分〜2分半)

**この動画のゴール**: 「お客様と同じ目線で見る」チェックが習慣になること。フォームのテスト送信の大切さを伝える。

| シーン | 画面に映すもの | ナレーション |
|---|---|---|
| オープニング(15秒) | タイトルカード | 最後の3本目は「サイトの健康チェック」。管理画面ではなく、お客様と同じ目線でサイトを見てあげる時間です。 |
| ステップ1(25秒) | スマホでトップページをスクロール | まず、スマホでトップページを開いて、上から下までゆっくりスクロール。文字や画像の崩れ、表示されない画像がないかを見ていきます。 |
| ステップ2(15秒) | PCでトップページを表示 | 同じことを、パソコンでも。ざっと全体を眺めるだけで大丈夫です。 |
| ステップ3(35秒) | フォームに「テスト」と入力→送信→受信メール | そして、いちばん見落としやすいのがこれ。お問い合わせフォームから、自分でテスト送信をしてみましょう。名前に「テスト」と書いて送り、通知のメールがちゃんと届けば合格です。フォームの故障は見た目では気づけないのに、お問い合わせを逃してしまう、いちばんもったいない故障です。月に1回、必ず試してください。 |
| 注意(30秒) | 見覚えのない広告・警告画面の例(イメージ図でOK) | 最後に、危険なサインを3つだけ覚えてください。「見覚えのない広告や英語のページが出る」「"このサイトは安全ではありません"という警告が出る」「テスト送信が届かない」。どれかひとつでも見つけたら、放置せず、気づいたその日にご連絡ください。 |
| 締め(20秒) | 連絡先カード→3本のサムネイルが並ぶ画面 | これで月イチ15分メンテはすべて完了です。来月もこの3本を順番に見ながら進めてください。(共通の締め定型文) |

## NotebookLM等で作る場合のメモ

- 読み込ませるソースは **backup-manual.pdf + このシナリオ集** の2点。「このシナリオの構成・ナレーションに沿って」と指示すると内容がぶれにくい
- 1本の動画に1作業だけ。3本まとめた長編は作らない(クライアントが「今月やる分」だけ再生できるのがこの構成の狙い)
- 生成後は必ず全編を再生して確認: ①価格や連絡先を勝手に言っていないか ②「〜と思われます」など曖昧な言い回しになっていないか ③専門用語(FTP・phpMyAdmin等)が混ざっていないか
- 完成した動画の格納先とクライアントへの渡し方(限定公開URL等)が決まったら、紙マニュアルv02で表紙・チェックリストに動画タイトルとリンクを載せる
